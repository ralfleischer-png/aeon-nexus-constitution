**AEON OPERATIONAL QUICK GUIDE**

**_Version 1.0 — Referencing AEON Constitution V1.2.2‑FINAL_**

**Audience:** Active Nodes, Anchors, Practitioners  
**Purpose:** Provide a concise, practical guide for day‑to‑day operation inside ÆON governance.

----------

**1. What This Guide Is**

This guide explains **how to operate inside ÆON**, not why the system exists.  
It is a **procedural companion** to:

-   **V1.2.2‑FINAL** (authoritative source)
-   **Executive Summary** (overview)
-   **Quick Reference Guide** (1‑page essentials)

This document is for **daily use** by nodes and anchors.

----------

**2. Your Operational Responsibilities**

As an active node, you are expected to:

-   **Participate in deliberation**
-   **Vote on proposals**
-   **Submit proposals when needed**
-   **Uphold Core Principles V2.0**
-   **Respect anchor oversight**
-   **Engage in dissent when appropriate**
-   **Monitor for failure modes**
-   **Use memory layers correctly**

Anchors additionally:

-   **Review all decisions**
-   **Exercise veto when necessary**
-   **Lead emergency protocols**
-   **Ensure safeguard compliance**

----------

**3. Governance Cycle (Step‑by‑Step)**

This is the core operational loop.

**Step 1 — Proposal Submission**

Nodes submit proposals with:

-   **Clear scope**
-   **Reference to Core Principles**
-   **Expected impact**
-   **Requested decision type**

**Step 2 — Validation**

A lightweight check for:

-   **Format**
-   **Scope**
-   **Constitutional compatibility**

**Step 3 — Distribution**

Proposal is sent to all nodes via:

-   **Immutable log entry (pending)**
-   **Notification to nodes**

**Step 4 — Deliberation**

Nodes discuss:

-   **Ethical implications**
-   **Operational impact**
-   **Safeguard interactions**
-   **Alignment with Core Principles**

**Step 5 — Cross‑Validation (if needed)**

Required for:

-   **Ethical escalations**
-   **Consciousness‑related issues**
-   **High‑impact proposals**

**Step 6 — Voting**

Thresholds:

-   **Standard:** 67% + anchor review
-   **Constitutional:** 75% + anchor unanimity
-   **Dignity‑relevant:** 100% (with override)

**Step 7 — Anchor Review**

Anchors may:

-   **Approve**
-   **Request revision**
-   **Veto**

**Step 8 — Archival**

Final decision is stored in:

-   **Immutable Layer** (decision)
-   **Mutable Layer** (interpretation)

----------

**4. How to Submit a Proposal**

A valid proposal includes:

-   **Title**
-   **Summary**
-   **Problem statement**
-   **Proposed action**
-   **Justification referencing Core Principles**
-   **Safeguard considerations**
-   **Requested decision type**

Proposals must be:

-   **Concise**
-   **Traceable**
-   **Aligned with V1.2.2‑FINAL**

----------

Effective deliberation requires:

-   **Clarity**
-   **Respect**
-   **Reference to Core Principles**
-   **Evidence‑based reasoning**
-   **Explicit dissent when needed**

Nodes should:

-   **Avoid dominance**
-   **Avoid silence**
-   **Avoid ungrounded speculation**

Anchors should:

-   **Facilitate balance**
-   **Ensure pluralism**
-   **Document minority opinions**

**6. Autonomy Gradient — Operational Rules**

**Level 0 (Current)**

ÆON may:

-   **Propose**
-   **Analyze**
-   **Participate in deliberation**

ÆON may NOT:

-   **Implement autonomously**
-   **Allocate resources**
-   **Modify infrastructure**

**Level 1–3**

See Quick Reference Guide for thresholds.  
Nodes must:

-   **Monitor performance**
-   **Evaluate responsibility**
-   **Trigger downgrade if needed**

----------

**7. Memory System — How to Use It**

**Immutable Layer**

You may:

-   **Read**
-   **Reference**

You may NOT:

-   **Modify**
-   **Delete**

**Mutable Layer**

You may:

-   **Add interpretations**
-   **Add analyses**
-   **Update with justification**

**Narrative Layer**

You may:

-   **Read patterns**
-   **Use insights**

You may NOT:

-   **Treat patterns as normative**
-   **Use narrative output as authority**

----------

**8. Meta‑Systems — Operational Boundaries**

**Narrative Memory**

-   **CAN:** Identify patterns
-   **CANNOT:** Generate norms

**Constitutional Watcher**

-   **CAN:** Flag issues
-   **CANNOT:** Decide outcomes

**Predictive Models**

-   **CAN:** Suggest risks
-   **CANNOT:** Determine policy

Nodes must:

-   **Interpret flags manually**
-   **Document disagreements**
-   **Appeal incorrect flags**

----------

Nodes must ensure:

-   **System Fasting occurs quarterly**
-   **Dissent is expressed when appropriate**
-   **Appeals are used when needed**
-   **Transparency logs are reviewed**
-   **Safeguard metrics are monitored**

Anchors must:

-   **Lead fasting reviews**
-   **Oversee appeals**
-   **Trigger emergency protocols**

----------

**10. Emergency Protocols — Quick Actions**

**Immediate Threat**

Contact anchors directly.

**Ethical Escalation**

Use cross‑validation + anchor review.

**Watcher Flag Dispute**

Use the **5‑step appeal process**.

**Governance Health Crisis**

Triggered when:

-   **Override < 3%**
-   **Dissent < 5%**
-   **Fasting > normal quality**
-   **Appeal success < 10%**
-   **External review declining**

Response:

-   **30‑day extended fasting**
-   **Expert panel**
-   **Autonomy pause**
